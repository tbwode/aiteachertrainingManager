### 1.3 适用范围

**覆盖功能边界**：
- ✅ 本 PRD 覆盖 AI 教师研训超管端（PC 管理端）全部功能模块，包括：AI 研训驾驶舱、研训课程管理、研训活动管理、课程目录管理（含 AI 智能导入）、系统配置（课程类型、能力维度）。

**不做什么**：
- 不覆盖移动端适配；不覆盖与外部教务系统、人事系统的数据对接；不覆盖 AI 模型的训练与调优逻辑。

## 二、需求背景与目标

### 2.1 业务背景

在基础教育（普教）领域，教师专业发展研训是提升教学质量的核心手段。当前研训管理存在以下痛点：
- **课程资源分散**：各类研训课程、活动分散在不同平台，缺乏统一的管理与分发入口。
- **数据收集困难**：教师学习记录、课程评价、反馈数据分散，无法形成有效闭环。
- **人工统计低效**：研训活动参与情况、学时统计、效果评估依赖人工汇总 Excel，效率低且易出错。
- **过程管理缺失**：活动发布后，管理员无法实时跟踪学员的学习进度、课程完成情况、反馈内容。

### 2.2 用户痛点

| 痛点 | 描述 | 量化影响 |
|------|------|----------|
| 课程管理成本高 | 课程信息（封面、类型、讲师、目录）需手动录入，课件需逐条添加 | 发布一门课程耗时 > 30 分钟 |
| 活动跟进困难 | 活动发布后无法实时掌握学员学习进度，需手动催促 | 逾期未完成学员无法及时识别 |
| 数据呈现不足 | 缺乏可视化数据看板，无法快速洞察研训整体效果 | 管理层决策依赖滞后报表 |
| 反馈收集低效 | 学员学习心得/反馈收集依赖线下或分散渠道，汇总困难 | 反馈收集率 < 50% |

### 2.3 产品目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|--------|
| 降低课程发布成本 | 单门课程录入时间（含目录） | 从 30 分钟降至 10 分钟（AI 智能导入辅助） |
| 提升活动管理效率 | 活动跟进实时数据覆盖率 | 100% 学员学习进度可实时查看 |
| 增强数据可视化能力 | 核心指标（课程数、活动数、参与率）大屏呈现 | 关键指标 3 秒内加载 |
| 提高反馈收集率 | 学习心得收集率 | 通过活动强制完成机制提升至 80% 以上 |

### 2.4 需求价值

| 价值维度 | 说明 |
|----------|------|
| 用户价值 | 管理员一站式完成课程/活动发布、学员跟进、数据查看，减少重复劳动 |
| 业务价值 | 实现研训全流程线上化、数据化，为教师发展决策提供数据支撑 |
| 产品价值 | 构建"内容运营-学习消费-数据追踪"的管理闭环，支撑产品持续迭代 |

## 三、用户角色与使用场景

| 角色 | 权限范围 | 原型对应功能 |

### 3.2 使用场景

#### 场景一：驾驶舱数据总览（P0）

| 维度 | 描述 |
|------|------|
| 场景背景 | 管理员登录系统后，需快速了解当前研训全局数据 |
| 用户操作 | 进入"AI研训驾驶舱"页面，查看人员总览、能力维度分布、学科排行、活动完成情况、热门排行、词云 |
| 期望结果 | 3 秒内获取核心指标，支持全屏展示，数据每 5 秒自动刷新（模拟实时感） |

#### 场景二：发布研训课程（P0）

| 维度 | 描述 |
|------|------|
| 场景背景 | 管理员需将新的研训课程上线供教师学习 |

| 维度 | 描述 |

| 维度 | 描述 |

## 五、核心业务流程

### 5.1 主业务流程一：发布研训课程

```
[管理员登录] → [进入研训课程列表] → [点击新建课程]
    → [填写基本信息：名称、封面、介绍、类型、部门、维度、讲师]
    → [基本信息校验通过] → [切换至课程目录 Tab]
    → [添加章 / 添加课件 / 使用 AI 智能导入批量导入]
    → [保存为草稿 或 保存并上架]
    → [返回课程列表，课程状态为"草稿"或"已上架"]
```

### 5.2 主业务流程二：发布研训活动

```
[管理员登录] → [进入研训活动列表] → [点击新建活动]
    → [填写基本信息：名称、封面、目标、时间、参与教师]
    → [配置学时机制：是否设置学时、需修满学时]
    → [添加课程：从课程库中选择，批量设置学时]
    → [设置完成学习方式：查看课程 / 完成课时 / 提交学习心得 / 完成+心得]
    → [保存并发布]
    → [返回活动列表，活动状态根据当前日期自动判定为"未开始"/"进行中"/"已结束"]
```

### 5.3 主业务流程三：活动跟进与数据追踪

```
[管理员进入活动列表] → [选择活动] → [点击活动跟进]
    → [查看学员学习情况 Tab]
        → [查看总览：已学习课程数、获得学时、完成状态]
        → [点击"查看详情"]
        → [弹窗展示：学时获得柱状图、课程学习柱状图、学习明细表格]
        → [可查看学习心得内容]
        → [导出 Excel]
    → [查看课程学习情况 Tab]
        → [查看每门课程的完成学习人数 / 参与人数]
        → [点击"查看详情"]
        → [弹窗展示：课件学习明细]
        → [导出 Excel]
```

### 5.4 异常流程

| 异常场景 | 触发条件 | 处理规则 | 提示文本 |
|----------|----------|----------|----------|
| 删除已上架课程 | 管理员点击删除，课程状态为 published | 禁止删除，提示先下架 | "请先下架课程再删除" |
| 删除非空章节 | 管理员点击删除章，章下存在课件 | 禁止删除 | "该章下已有内容，不支持删除" |
| 删除已关联类型 | 管理员删除课程类型，已有课程关联该类型 | 禁止删除 | "该类型下已有课程关联，无法删除" |
| 删除已关联维度 | 管理员删除能力维度，已有课程关联该维度 | 禁止删除 | "该维度已被课程关联，无法删除" |
| 学时超限 | 活动需修满学时 > 课程学时之和 | 保存失败 | "需修满学时不能大于课程学时之和（{total} 学时）" |
| 未选择参与教师 | 活动保存时未选择教师 | 保存失败 | "请选择参与教师" |
| 未上传封面 | 课程/活动保存时未上传封面 | 保存失败 | "请上传课程封面" / "请上传活动封面" |
| 拖拽跨章排序 | 用户尝试将课件拖拽至不同章节 | 静默忽略，无反馈 | 无提示（当前实现） |
| 活动时间逆序 | 开始时间 > 结束时间 | 无校验，允许保存 | 需补充校验 |
| 重复添加课程 | 同一课程被多次添加至活动 | 允许重复，无提示 | 需补充校验 |

## 六、功能详细说明

> 【研发重点关注】【测试重点关注】本章严格拆解原型的每个页面、每个功能、每个字段，确保与原型 100% 一致。

### 6.1 AI 研训驾驶舱

#### 6.1.1 功能描述
为管理员提供研训全局数据的实时可视化大屏，覆盖人员、课程、活动、能力、排行等核心维度，支持全屏展示与自动刷新。

#### 6.1.2 前置条件
- 管理员已登录系统
- 系统已存在课程、活动、教师数据（允许空数据展示）

#### 6.1.3 入口
- 左侧导航栏 → AI 研训 → 研训驾驶舱

#### 6.1.4 页面布局
```
┌─────────────────────────────────────────────────────┐
│  [Logo]  AI教师研训    [全屏按钮]       [实时时间] │  ← 顶部标题栏
├──────────┬────────────────────────────┬─────────────┤
│ 人员总览  │    核心指标卡（3列）        │ 热门课程 TOP5│
│ 能力雷达  │  活动参与完成情况（堆叠柱状）│ 活跃教师 TOP5│
│ 学科排行  │  教师学习时长趋势（折线）    │ 热门活动 TOP5│
│          │  高频词云                   │ 热门问题 TOP5│
└──────────┴────────────────────────────┴─────────────┘
```

#### 6.1.5 操作流程
1. 管理员点击左侧导航"研训驾驶舱"
2. 页面加载，数字指标以滚动动画呈现（800ms 内从 0 滚动至目标值）
3. 图表自动渲染（ECharts 初始化）
4. 每 5 秒自动刷新数据（模拟实时感，数据微变）
5. 点击"全屏"按钮进入全屏模式，再次点击退出

#### 6.1.6 交互状态
| 元素 | 默认态 | Hover 态 | 点击态 | 禁用态 |
|------|--------|----------|--------|--------|
| 全屏按钮 | 白底黑字，图标+文字"全屏" | 背景色加深 | 切换全屏/退出 | 无 |
| 热门排行项 | 常规文本，进度条 | 光标变手型 | 跳转至对应详情页 | 无 |
| 实时时间 | 每秒更新 | 无 | 无 | 无 |

#### 6.1.7 页面字段定义

**左侧栏 - 人员总览**

| 字段名 | 输入类型 | 最大长度 | 是否必填 | 默认值 | 枚举选项 | 格式规则 | 计算逻辑 |
|--------|----------|----------|----------|--------|----------|----------|----------|
| 教师参训总人数 | 数字展示 | - | 是 | 0 | - | 整数 | 所有研训活动中参与教师数量（去重，含已完成与未完成） |
| 完成人数 | 数字展示 | - | 是 | 0 | - | 整数 | 所有活动中 `completed=true` 的教师数（去重） |
| 研训完成率 | 百分比展示 | - | 是 | 0% | - | XX% | 完成人数 / 参训总人数 × 100%，四舍五入 |

**左侧栏 - 教师能力维度分布（雷达图）**

| 字段名 | 说明 |
|--------|------|
| 维度名称 | 能力维度名称（从能力维度管理模块获取），取学校教师能力维度平均分最高的前 6 个维度展示 |
| 维度最大值 | 100 |
| 维度平均分 | 该维度下所有教师评分的平均值，按降序排列后取前 6 名 |

> **计算逻辑**：
> 1. 遍历学校所有教师，获取每位教师在各能力维度下的评分；
> 2. 对每个维度，计算所有教师评分的算术平均值；
> 3. 按平均分降序排序，取排名前 6 的维度；
> 4. 若维度总数不足 6 个，则展示全部维度。

**左侧栏 - 学科参训分布排行（横向柱状图）**

| 字段名 | 说明 |
|--------|------|
| 学科名称 | 教师学科字段（如"数学"、"语文"） |
| 参训人数 | 该学科下参与研训的教师数 |

**中间栏 - 核心指标卡（3 列）**

| 字段名 | 图标颜色 | 计算逻辑 |
|--------|----------|----------|
| 研训课程数 | #C8000C（红） | 课程总数 |
| 研训活动数 | #78716C（灰） | 活动总数 |
| 能力维度平均分 | #A8A29E（浅灰） | 所有维度平均分的平均值，四舍五入 |

**中间栏 - 活动参与完成情况（堆叠柱状图）**

| 字段名 | 说明 |
|--------|------|
| 活动名称 | 按活动开始时间降序取最新的 10 个活动名称（去除括号及内容） |
| 已完成 | 该活动中 `completed=true` 的教师数 |
| 未完成 | 该活动中总参与人数 - 已完成数 |

> **排序与展示规则**：
> 1. 按活动开始时间 `startTime` 降序排列；
> 2. 取时间最新的前 10 个活动展示；
> 3. 若活动总数不足 10 个，则展示全部活动。

**中间栏 - 教师学习时长趋势（折线图）**

| 字段名 | 说明 |
|--------|------|
| 日期 | 学习记录完成时间的月份-日期（MM-DD） |
| 学习时长 | 该日期下所有学习记录的 `hours` 之和 |

> **日期展示范围逻辑**：
> 1. 展示时间范围为最近 7 天（含当天）；
> 2. 横轴日期按自然日连续排列，格式为 "MM-DD"；
> 3. 若某日期无学习记录，该日期点仍展示，学习时长为 0；
> 4. 日期范围以当前系统日期为基准，向前追溯 6 天，共 7 个点。"

**中间栏 - 高频词云**

| 字段名 | 说明 |
|--------|------|
| 关键词 | 从用户 AI 研训助手的提问问题中提取关键词，按频次降序展示前 20 个 |
| 词频 | 该关键词在所有提问中的出现次数，决定字体大小（12px-32px） |
| 颜色 | 随机从 #C8000C、#E85D6A、#FAFAF9、#A8A29E、#FDE8E8 中选取 |

> **关键词提取逻辑**：
> 1. 采集用户向 AI 研训助手发送的所有提问文本；
> 2. 使用中文分词（如 jieba）对提问文本进行分词，去除停用词（"的、了、是、怎么"等）；
> 3. 统计每个词在全部提问中的出现频次；
> 4. 按频次降序排列，取前 20 个高频词展示；
> 5. 若提问总数不足或分词后有效词不足 20 个，则展示全部有效词。

**右侧栏 - TOP5 排行**

| 排行项 | 排序依据 | 取值字段 |
|--------|----------|----------|
| 热门课程 TOP5 | 学习人数降序 | `course.learners` |
| 活跃教师 TOP5 | 获得学时降序 | 所有活动中该教师的 `earnedHours` 之和 |
| 热门活动 TOP5 | 参与人数降序 | `activity.participants.length` |
| 热门问题 TOP5 | 提问次数降序 | 用户 AI 研训助手的提问问题原文，取提问次数最高的前 5 个问题展示 |

> **热门问题计算逻辑**：
> 1. 采集用户向 AI 研训助手发送的所有提问问题（按问题原文去重）；
> 2. 统计每个问题的提问次数（相同问题多次发送，次数累加）；
> 3. 按提问次数降序排列，取前 5 个问题展示；
> 4. 展示内容为问题原文，可截断显示（超出 20 字显示省略号）；
> 5. 若不同问题不足 5 个，则展示全部。

#### 6.1.8 状态流转
无复杂状态流转，数据均为实时计算展示。

#### 6.1.9 异常与边界
| 异常场景 | 处理规则 |
|----------|----------|
| 无数据 | 所有指标显示 0，图表渲染空数据 |
| 词云数据为空 | 渲染空白区域 |
| 全屏 API 不支持 | 静默失败，按钮状态不变 |

#### 6.5.6 AI 匹配算法（写入 PRD）

**算法核心**：通过 AI 分析课件内容，结合章节名称进行智能匹配，实现课件与已有章节的精准关联。

**算法输入**：上传文件列表（文件名、文件内容、类型）、当前课程已有章节列表（`tempCatalog`）

**算法步骤**：
1. **AI 内容分析**：提取课件文件中的核心知识点、主题关键词
   - 读取课件文本内容（支持 Word、PDF、PPT 等格式）
   - 提取标题、摘要、章节小结等关键文本片段
   - 通过 AI 语义分析，生成课件内容特征向量
2. **章节名称语义匹配**：将课件内容特征与已有章节名称进行语义相似度计算
   - 计算课件内容关键词与章节名称的文本相似度
   - 语义关联度：内容主题与章节名称的语义匹配 → +80 分
3. **文件名辅助匹配**（作为补充校验）：
   - **提取章节号**：从文件名中提取章节编号
     - 匹配规则：`第\s*([一二三四五六七八九十0-9]+)\s*章`
     - 匹配规则：`Chapter\s*([0-9]+)`（不区分大小写）
     - 匹配规则：`^\s*([0-9]+)[\.\-_]`
     - 中文数字转换：一=1, 二=2, ..., 十=10（处理"十"为 10，"十三"为 13 等）
   - **提取核心名称**：去除章节号前缀后的文件名主体
     - 去除：`第\s*[一二三四五六七八九十0-9]+\s*章[\.\-_\s]*`
     - 去除：`Chapter\s*[0-9]+[\.\-_\s]*`（不区分大小写）
   - 已有章节匹配评分（辅助维度）：
     - 章节号匹配：文件章节号 = 已有章节号 → +100 分
     - 核心名称包含：已有章节核心名称 ⊆ 文件核心名称 → +50 分
     - 反向包含：文件核心名称 ⊆ 已有章节核心名称 → +40 分
     - 精确匹配：文件核心名称 = 已有章节核心名称 → +30 分
4. **综合匹配判定**：
   - 综合 AI 语义匹配分（权重 60%）与文件名辅助匹配分（权重 40%）
   - 取综合得分最高章节，若最高分 ≥ 50，则匹配至该章节；否则视为新章节
5. **新章节命名**（未匹配时）：
   - 优先使用 AI 从课件内容中提取的核心主题作为章节名称
   - 若无法提取，则使用文件名中的章节号标识（如"第三章"、"Chapter 3"、"3."）
   - 无章节号时，默认命名为"导入内容"
6. **课件名称**：基于 AI 内容分析提取的标题，或核心名称（去除首尾标点、空格）

**算法输出**：每个文件对应的章节名称（新/已有）、课件名称、匹配章节 ID（如有）

| 学习次数 | 该课件的学习次数 | 模拟计算：`learnedCount + (index % 3)` |

## 十、风险与应对方案

### 10.1 业务风险

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 课程/活动数据误删除 | 教师学习记录、学时数据丢失 | 删除前二次确认（活动删除需明确提示学时清除）；后续增加回收站机制 |
| 活动状态计算不准确 | 管理员对活动状态的判断出现偏差 | 状态由服务端根据日期计算，避免客户端时区差异；增加状态说明提示 |
| 学时统计与实际学习不一致 | 教师对学时获得有异议 | 学时计算规则明确写入 PRD，记录详细学习日志，支持按课件追溯 |
| 学习心得内容审核缺失 | 出现不当言论 | 后续增加敏感词过滤和人工审核机制 |

### 10.2 技术风险

| 风险 | 影响 | 规避措施 |
|------|------|----------|
| 富文本编辑器 XSS 漏洞 | 存储型 XSS 攻击 | 后端对富文本内容进行标签白名单过滤，仅允许安全标签 |
| 文件上传安全风险 | 恶意文件上传 | 限制文件类型、大小，服务端二次校验 MIME 类型，禁止执行文件 |
| 驾驶舱数据量过大 | 图表渲染卡顿 | 数据聚合在服务端完成，前端仅接收聚合结果；大数据量时分段加载 |
| 并发编辑冲突 | 数据覆盖 | 后续增加乐观锁机制（版本号控制），编辑时检测数据变更 |
| AI 导入匹配不准确 | 用户手动调整成本高 | 算法持续迭代，增加更多命名规则支持；提供清晰的手动调整界面 |

### 10.3 上线风险

| 风险 | 影响 | 回滚方案/应急处理 |
|------|------|-------------------|
| 活动数据异常 | 活动状态、学时统计错误 | 提供数据修复脚本，支持手动修正活动状态和学时；保留数据备份 |
| 课程类型/维度数据丢失 | 课程关联关系断裂 | 上线前全量备份；删除操作改为软删除，支持恢复 |
| 性能不达标 | 页面加载慢、操作卡顿 | 预上线压测；如不达标，延后驾驶舱实时刷新功能，改为手动刷新 |
| 文件存储服务异常 | 封面、课件无法上传 | 降级为本地存储或关闭上传功能，优先保障核心业务流程 |
