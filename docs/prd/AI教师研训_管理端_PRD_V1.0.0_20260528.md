## 二、需求背景与目标

### 2.1 业务背景
在教育数字化转型背景下，各级教育机构需要系统化地组织教师培训活动，涵盖师德师风、教学能力、信息技术、心理健康等多个维度。当前多数机构仍采用线下培训或分散的线上培训方式，存在以下问题：
- 培训课程缺乏统一管理，内容重复、质量参差不齐
- 培训活动组织效率低，人工统计学习进度耗时久
- 教师学习数据分散，无法形成完整的培训档案
- 培训效果难以量化评估，缺乏数据支撑决策

### 2.2 用户痛点

| 痛点 | 具体描述 |
|------|----------|
| 课程管理混乱 | 培训课程分散在不同平台，无统一分类体系，查找困难 |
| 活动组织低效 | 创建培训活动需人工通知、人工统计参与情况，耗时长 |
| 学习进度不透明 | 无法实时查看教师学习情况，逾期学员难以识别 |
| 数据导出困难 | 学习数据、统计报表需人工汇总，无法一键导出 |
| 课件维护繁琐 | 课程目录结构变更困难，课件增删改操作复杂 |

### 2.3 产品目标

| 目标 | 量化指标 |
|------|----------|
| 提升课程管理效率 | 课程创建/编辑操作时间从30分钟降低至10分钟 |
| 提升活动组织效率 | 培训活动创建时间从2小时降低至30分钟 |
| 实现学习数据可视化 | 支持实时查看教师学习进度、课程学习数据 |
| 支持数据导出 | 所有列表均支持Excel导出，满足数据归档需求 |

### 2.4 需求价值

| 价值维度 | 说明 |
|----------|------|
| 对用户（管理员） | 提供一站式培训管理工具，降低管理成本，提升工作效率 |
| 对业务（教育机构） | 规范培训流程，沉淀培训资源，形成可复用的培训体系 |
| 对产品 | 完善平台核心能力，形成课程-活动-学习的数据闭环 |

## 三、用户角色与使用场景

### 3.1 用户角色

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 可访问所有功能模块：培训课程管理、培训活动管理、课程类型管理、数据查看与导出 |

### 3.2 场景描述

#### 场景1：创建培训课程（P0）
- **场景背景**：管理员需要新增一门师德师风类培训课程
- **用户操作**：进入培训课程列表 → 点击"新建课程" → 填写课程基本信息 → 配置课程目录 → 保存课程
- **期望结果**：课程创建成功，状态为"草稿"，可后续编辑或上架

#### 场景2：上架/下架课程（P0）
- **场景背景**：课程审核通过后需要上架供教师学习，或课程过期需要下架
- **用户操作**：课程列表 → 找到目标课程 → 点击"上架"/"下架" → 确认操作
- **期望结果**：课程状态变更，上架后教师端可见，下架后教师端不可见

#### 场景3：创建培训活动（P0）
- **场景背景**：暑期需要组织一次教师专业能力提升培训
- **用户操作**：进入培训活动列表 → 点击"新建活动" → 填写活动信息 → 选择参与教师 → 添加课程 → 设置学时机制 → 保存活动
- **期望结果**：活动创建成功，参与教师可在教师端看到该活动

#### 场景4：跟进活动学习情况（P1）
- **场景背景**：活动进行中，管理员需要查看教师学习进度
- **用户操作**：活动列表 → 点击"活动跟进" → 切换"学员学习情况"/"课程学习情况"Tab → 查看数据/导出Excel
- **期望结果**：实时展示学员学习进度和课程学习人数，支持数据导出

#### 场景5：查看课程数据（P1）
- **场景背景**：需要了解某门课程的受欢迎程度和学习效果
- **用户操作**：课程列表 → 点击"课程数据" → 查看章节详情/教师学习详情 → 导出数据
- **期望结果**：展示课程整体学习数据、各课件学习情况、每位教师的学习进度

#### 场景6：管理课程类型（P1）
- **场景背景**：新增了一个培训方向，需要添加对应的课程类型
- **用户操作**：课程列表 → 点击"管理类型" → 添加根类型/添加下级 → 保存
- **期望结果**：类型添加成功，创建课程时可选择该类型

### 3.3 场景优先级

| 优先级 | 场景 |
|--------|------|
| P0 | 创建培训课程、上架/下架课程、创建培训活动 |
| P1 | 跟进活动学习情况、查看课程数据、管理课程类型 |
| P2 | 导出各类数据报表 |

## 五、核心业务流程

### 5.1 主业务流程：创建培训课程

1. 管理员进入"培训课程"列表页
2. 点击"新建课程"按钮，进入课程表单页
3. 填写基本信息Tab：课程名称、课程封面、课程介绍、课程类型、课程提供部门、课程讲师/机构
4. 切换到课程目录Tab：添加章/课件，配置课件信息
5. 点击"保存草稿"或"直接上架"
6. 系统校验必填字段，校验通过后保存数据
7. 返回课程列表页，展示新建/更新后的课程

### 5.2 主业务流程：创建培训活动

1. 管理员进入"培训活动"列表页
2. 点击"新建活动"按钮，进入活动表单页
3. 填写基本信息：活动名称、活动封面、培训目标、培训目标附件、开始时间、结束时间、参与教师
4. 配置学时机制：是否设置学时、需修满学时
5. 添加课程内容：选择课程、设置可得学时、选择完成学习方式
6. 点击"保存"
7. 系统校验必填字段和学时逻辑，校验通过后保存数据
8. 返回活动列表页

### 5.3 核心子流程：活动跟进查看

1. 管理员进入"培训活动"列表页
2. 找到目标活动，点击"活动跟进"
3. 进入活动跟进页，默认展示"学员学习情况"Tab
4. 可切换至"课程学习情况"Tab
5. 点击"查看详情"弹出学员详情弹窗或课程学习详情弹窗
6. 可导出Excel数据

### 5.4 异常流程

| 异常场景 | 触发条件 | 处理规则 |
|----------|----------|----------|
| 删除已上架课程 | 点击删除按钮时课程状态为published | 提示"已上架课程无法删除，请先下架"，终止删除 |
| 上架无内容课程 | 点击上架时课程目录为空 | 提示"课程目录不能为空，请添加至少一个课件"，终止上架 |
| 学时不匹配 | 需修满学时大于课程学时之和 | 提示"需修满学时不能大于课程学时之和（X学时）"，终止保存 |
| 删除有关联课程的类型 | 删除类型时其下存在关联课程 | 提示"该类型下存在关联课程，无法删除"，终止删除 |
| 删除有内容的章 | 删除章时其下存在课件 | 提示"该章节下存在课件，无法删除"，终止删除 |
| 编辑活动限制字段 | 编辑时修改开始时间/学时开关/学习方式 | 字段置灰不可编辑，hover提示"创建后不可修改" |

## 六、功能详细说明

### 6.1 培训课程列表（功能编号：C-001）

#### 6.1.1 功能描述
展示所有培训课程的列表，支持搜索、筛选、分页，提供新建课程、管理类型入口，以及每门课程的编辑、上下架、课程数据、删除操作。

#### 6.1.2 前置条件
- 用户已登录且拥有超级管理员权限
- 课程类型数据已初始化

#### 6.1.3 入口
- 左侧菜单：培训管理 → 培训课程
- 页面路径：`/course-list`

#### 6.1.4 页面布局

```
页面顶部：面包屑（培训课程）+ 页面标题 + 副标题 + 操作按钮区
├─ 左侧：管理类型按钮
└─ 右侧：新建课程按钮

页面主体：卡片容器
├─ 筛选栏（filter-bar）
│  ├─ 搜索框（课程名称）
│  ├─ 类型多选下拉
│  ├─ 状态下拉
│  ├─ 开始日期选择
│  ├─ "至"文本
│  └─ 结束日期选择
├─ 表格区（table-wrapper）
└─ 分页区（pagination）
```

#### 6.1.5 操作流程

1. 进入页面，默认展示全部课程，按更新时间倒序排列
2. 在搜索框输入课程名称关键词，实时筛选列表
3. 点击类型多选下拉，选择课程类型（支持搜索、多选、清空、确定）
4. 选择状态下拉（全部/已上架/草稿/已下架）
5. 选择时间范围（开始日期-结束日期）
6. 点击"新建课程"进入课程表单页
7. 点击"管理类型"打开课程类型管理抽屉
8. 点击课程行操作按钮：编辑/上下架/课程数据/删除

#### 6.1.8 新增与编辑的字段差异
本功能为列表页，无新增/编辑表单。

#### 6.1.9 状态流转

```
草稿 → 上架 → 已上架
已上架 → 下架 → 已下架
已下架 → 上架 → 已上架
草稿/已下架 → 删除 → 已删除
```

**状态流转规则：**
- 草稿状态：可编辑、可上架、可删除
- 已上架状态：可编辑、可下架、不可删除
- 已下架状态：可编辑、可上架、可删除

#### 6.2.5 操作流程

3. 上传课程封面（点击上传区域，选择图片文件）
4. 填写课程介绍（使用富文本编辑器）
5. 选择课程类型（点击下拉，支持搜索、多选、清空、确定）

14. 在同一章下，长按课件左侧拖拽手柄，拖拽课件可调整顺序
15. 点击"保存草稿"或"直接上架"
16. 系统校验必填字段和目录非空，通过后保存并返回列表

**编辑课程：**
1. 点击"编辑"进入表单页，自动填充已有数据
2. 在"基本信息"Tab修改基础信息，或在"课程目录"Tab增删改章和课件
3. 点击"保存草稿"或"直接上架"

#### 6.3.4 视频课件

##### 6.3.4.1 业务场景
以视频形式呈现课程内容，适用于课堂实录、专家讲座、操作演示、微课等场景。

##### 6.3.4.2 专属字段

| 字段名 | 输入类型 | 最大长度 | 是否必填 | 默认值 | 格式规则 |
|--------|----------|----------|----------|--------|----------|
| 视频文件 | 文件上传 | 无 | 是 | 无 | 支持mp4、mov、avi、wmv、flv、mkv；单个文件不超过2GB；建议分辨率1080p及以上；上传后生成封面截图（取视频第5秒帧） |
| 播放设置 | 开关组 | 无 | 否 | 全部关闭 | ① 禁止拖拽进度条（开启后教师端不可快进）；② 禁止倍速播放（开启后教师端不可调整播放速度） |

##### 6.3.4.3 已完成判定规则
教师端视频播放进度达到**视频末尾**（进度条播放到最后），即视为"已完成"。
若开启"禁止拖拽进度条"，则按实际观看时长计算；若关闭，则按播放进度计算。

##### 6.3.4.4 已学规则说明（展示文本）
> 教师观看该视频至进度条最后即视为已完成学习。若管理员设置了播放限制，教师需按正常播放速度完整观看，不可快进或倍速。

##### 6.3.4.5 下载设置
视频课件支持"允许下载"开关：
- **关闭**（默认）：教师端仅可在线观看，不可下载视频文件
- **开启**：教师端显示"下载视频"按钮，可下载原始视频文件到本地

##### 6.3.4.6 教师端呈现
- 列表页：显示"视频"类型标签
- 详情页：顶部显示课件名称，中间为视频播放器（支持全屏、音量调节、进度条）
- 播放器下方显示视频时长、当前观看进度百分比
- 若开启播放限制，进度条不可拖拽，倍速按钮隐藏
- 若允许下载，显示"下载视频"按钮
- 底部显示"已完成/未完成"状态标记

##### 6.3.4.7 异常与边界

| 异常场景 | 处理规则 |
|----------|----------|
| 视频文件未上传 | 保存时提示"请上传视频文件" |
| 视频格式不支持 | 上传时提示"仅支持mp4、mov、avi、wmv、flv、mkv格式" |
| 视频超过2GB | 上传时提示"视频文件不能超过2GB" |

| 教师端网络异常 | 已缓存的视频进度本地保存，恢复网络后同步学习记录 |

#### 6.3.5 文档课件

##### 6.3.5.1 业务场景
以文档形式呈现课程内容，适用于教案模板、教学设计、课例分析、培训手册、政策文件等场景。

##### 6.3.5.2 专属字段

| 字段名 | 输入类型 | 最大长度 | 是否必填 | 默认值 | 格式规则 |

| 转码状态 | 状态标签 | 无 | 否 | 转码中 | 上传后显示：转码中 → 转码成功/转码失败；转码失败时提示重新上传 |

教师端打开文档课件详情页，在线预览加载完成即视为"已完成"，无需浏览到文档末尾。
若文档转码失败，教师端提示"文档暂无法预览，请联系管理员"，该课件不计入已完成。

##### 6.3.5.4 已学规则说明（展示文本）
> 教师打开该文档课件页面即视为已完成学习。文档支持在线预览和下载（若管理员允许下载）。

##### 6.3.5.5 下载设置
文档课件支持"允许下载"开关：
- **关闭**（默认）：教师端仅可在线预览，不可下载原始文档
- **开启**：教师端显示"下载文档"按钮，可下载原始文档到本地

##### 6.3.5.6 教师端呈现
- 列表页：显示"文档"类型标签
- 详情页：顶部显示课件名称，中间为文档在线预览区
  - PDF：使用浏览器原生预览或PDF.js渲染
  - Word/Excel/PPT：转码后使用在线文档预览组件展示
- 右侧或底部显示文档页数/幻灯片数量信息
- 若允许下载，显示"下载文档"按钮
- 底部显示"已完成/未完成"状态标记

##### 6.3.5.7 异常与边界

| 文档超过100MB | 上传时提示"文档大小不能超过100MB" |

#### 6.8.5 操作流程

**新建活动：**
1. 点击"新建活动"进入表单页
2. 填写培训活动名称（最多30字符）
3. 上传活动封面（点击上传区域，选择图片文件）
4. 填写培训目标（最多200字符，可选）
5. 上传培训目标附件（支持多文件上传，可删除）
6. 选择开始时间和结束时间
7. 点击"选择参与教师"，在弹窗中选择教师（支持组织架构筛选、搜索、分页、全选）
8. 配置学时机制：开启或关闭学时设置
   - **学时开关关闭**：
     - 隐藏"需修满学时"输入框
     - 课程内容区域不显示"可得学时"列
     - 教师完成课程后不记录学时
   - **学时开关开启**：
     - 显示"需修满学时"输入框（必填，保留1位小数）
     - 课程内容区域显示"可得学时"列，可为每门课程设置学时
     - 显示"批量设置学时"按钮，可统一设置所有课程学时
     - 显示"总学时"统计（所有课程可得学时之和）
     - 保存时校验：需修满学时 ≤ 课程学时之和，否则提示"需修满学时不能大于课程学时之和（X学时）"
9. 点击"添加课程"，在弹窗中选择课程（支持课程体系筛选、搜索、分页、全选）
10. 设置每门课程的可得学时（仅学时开关开启时显示）
11. 点击"批量设置学时"，统一设置所有课程学时（仅学时开关开启时可用）
12. 选择完成学习方式（查看课程/完成课程下所有课件学习/提交反馈）
13. 点击"保存"
14. 系统校验必填字段和学时逻辑，通过后保存并返回列表

**编辑活动：**
1. 点击"编辑"进入表单页，自动填充已有数据
2. 修改允许修改的字段
3. 点击"保存"

## 十、风险与应对方案

### 10.2 业务风险

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 课程类型体系频繁变更 | 影响课程分类和筛选 | 提供类型管理功能，支持动态调整；类型变更时提醒管理员关联课程影响 |
| 活动删除后学时数据丢失 | 影响教师培训档案完整性 | 删除前二次确认，提示学时清除影响；考虑增加活动归档功能替代删除 |
| 课程上下架影响正在进行的活动 | 活动中的课程下架后教师无法学习 | 下架时校验课程是否被活动引用，引用时提示并阻止下架 |

### 10.3 技术风险

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 富文本编辑器XSS攻击 | 安全风险 | 前端过滤危险标签，后端二次过滤和转义 |
| 文件上传安全风险 | 恶意文件上传 | 限制文件类型和大小，服务端扫描病毒，存储至独立文件服务 |
| 大数据量列表性能问题 | 页面卡顿 | 后端分页查询，前端虚拟滚动，搜索走索引 |
| 并发编辑冲突 | 数据覆盖 | 增加乐观锁机制，编辑时检测数据版本 |

### 10.4 上线风险

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 数据迁移异常 | 历史数据丢失 | 提前备份数据，迁移脚本灰度执行，验证后全量执行 |
| 功能异常影响教师端 | 教师无法正常学习 | 管理端和教师端独立部署，管理端异常不影响教师端 |
| 性能不达标 | 用户体验差 | 上线前压测，预留扩容方案，监控告警 |
